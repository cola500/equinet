# S17-1/2/3: Speed Insights + WAF + robots/sitemap

Tre oberoende quick wins som alla kan göras i en branch.

## S17-1 Speed Insights

**Filer:**
- `src/app/layout.tsx` (ändra -- lägg till `<SpeedInsights />` bredvid `<Analytics />`)
- `package.json` (ändra -- ny dependency)

**Steg:**
1. `npm install @vercel/speed-insights`
2. Importera `SpeedInsights` från `@vercel/speed-insights/next`
3. Lägg till `<SpeedInsights />` efter `<Analytics />` i layout.tsx

**Verifiering:** Typecheck passerar. Fungerar i prod efter deploy.

## S17-2 WAF Custom Rules

**Filer:**
- `docs/operations/deployment.md` (ändra -- dokumentera reglerna)

**Steg:**
1. Dokumentera rekommenderade regler i deployment.md
2. Johan konfigurerar i Vercel Dashboard (kräver manuell åtgärd)

Regler att konfigurera:
- Regel 1: Blockera kända bot user-agents (scanner/scraper)
- Regel 2: Rate limit på `/api/auth/*` (extra lager)
- Regel 3: Geo-block utanför EU (om önskvärt)

**OBS:** Vercel Firewall konfigureras i Dashboard, inte i kod. Vi dokumenterar
reglerna och lämnar konfigurationen till Johan.

## S17-3 robots.txt + sitemap

**Filer:**
- `src/app/robots.ts` (ny -- Next.js metadata API)
- `src/app/sitemap.ts` (ny -- Next.js metadata API)

**Steg:**
1. Skapa `robots.ts` med Next.js Metadata API (blockera /api/, /admin/, /provider/)
2. Skapa `sitemap.ts` med statiska publika sidor
3. Verifiera med typecheck

**Approach:** Använder Next.js inbyggda metadata-filer istället för statiska
filer i `public/`. Ger typesafe och dynamisk generering.

## Risker

- **S17-1:** Ingen risk, rent additiv ändring
- **S17-2:** Kräver manuell Dashboard-åtgärd av Johan
- **S17-3:** Låg risk, nya filer utan sidoeffekter
