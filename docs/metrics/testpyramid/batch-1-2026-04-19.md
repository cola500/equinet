## Lärdomar

### Mönster etablerade (från S43-1 + S43-2)

1. **`vi.hoisted()` är obligatoriskt** för alla mock-objekt som refereras i `vi.mock()`-factories. Utan detta: `ReferenceError: Cannot access before initialization`.

2. **`withApiHandler`-routes mockar `@/lib/auth-dual`** (inte `@/lib/auth-server`). Gamla routes mockar `@/lib/auth-server`. Kontrollera alltid vilken auth-modul routen importerar.

3. **`as never` i mock-returvärden** ersätter `as any`. Universellt mönster i Vitest.

4. **`RateLimitServiceError`** måste inkluderas i `vi.mock('@/lib/rate-limit')` för `withApiHandler`-routes. Annars TypeError vid import.

5. **Befintlig route.test.ts täcker redan integration-testscenariot** (customer-insights, customer-registry): när route.test.ts redan importerar route-handler direkt och mockar bara gränser, skapa minimal smoke (3-4 tester) istället för att duplicera.

6. **Domänlogik kör för real**: DueForServiceCalculator och business-insights KPI-beräkningar kör utan mock — det ger genuint mer värde än att mocka allt.

### Gotchas

- **`groupBy` mock**: `prisma.booking.groupBy` returnerar `[]` som default — mock med `mockResolvedValueOnce` för det första anropet (stats) om routen gör flera `groupBy`-anrop.
- **`setCachedProviderInsights` är fire-and-forget**: Använd `vi.waitFor(() => expect(mock).toHaveBeenCalled())` vid verifiering — synkron assertion missar det.
