## Pilot 1 — unsubscribe (integration)

### Vad E2E-spec testade

- Giltig HMAC-länk → framgångssida "Du har avregistrerad dig"
- Ogiltig token → felmeddelande
- Saknade parametrar → felmeddelande
- Token utan userId → felmeddelande

### Vad integration-testet täcker

Samma 4 scenarios + 2 extra:
- Saknad token (bara userId)
- Rate-limiting → 429

### Genomförande

1. Lade till `vi.hoisted()` för mock-objekt (klassisk hoist-fälla med `vi.mock`)
2. Importerade `generateUnsubscribeToken` direkt från `@/lib/email/unsubscribe-token` — ingen E2E-utils nödvändig
3. Satte `process.env.NEXTAUTH_SECRET` i `beforeEach` för HMAC-generering

### Gotcha: vi.hoisted() obligatoriskt

`const mockX = { fn: vi.fn() }` före `vi.mock()` ger `ReferenceError: Cannot access 'mockX' before initialization` pga hoisting. Lösning:

```ts
const { mockX } = vi.hoisted(() => ({ mockX: { fn: vi.fn() } }))
vi.mock('@/lib/X', () => ({ X: mockX }))
```

## Pilot 2 — horses (component)

### Vad E2E-spec testade

- Sida laddar med heading och knapp
- Lägg till häst (formulär fill + submit → toast + dialog stängs)
- Redigera häst (navigering till detaljsida — inte HorseForm-specifikt)
- Ta bort häst med bekräftelse
- Navigering till hästdetalj

### Vad component-testet täcker

HorseForm-specifik coverage:
- Alla 8 fält renderas
- Namn-fält är required
- Submit-knapp inaktiv vid tomt namn / blanksteg-namn
- Submit-knapp aktiv vid ifyllt namn
- Inaktiv + "Sparar..."-text under isSaving
- setFormData anropas vid namnändring och rasändring
- onSubmit anropas vid klick
- Fördatafyllning visas korrekt i alla fält

### Genomförande

**Pre-pilot-fynd verifierat:** HorseForm var intern komponent i `page.tsx` — extraherades till `src/components/horses/HorseForm.tsx` med exporterade typer (`HorseFormData`, `emptyHorseForm`).

**Mock-strategi:**

```ts
vi.mock("@/components/ui/responsive-dialog", async (importOriginal) => {
  const orig = await importOriginal<...>()
  return {
    ...orig,
    ResponsiveDialogFooter: ({ children }) => <div>{children}</div>,
  }
})
```

`ResponsiveDialogFooter` renderar inget synligt i JSDOM utan denna stub.

### Täckning ej med i component-test (korrekt att skippa)

- Ta bort häst → `ResponsiveAlertDialog` med confirm — bör testas via `CustomerHorsesPage`-test eller behålls i integration
- Navigering till hästdetalj → browser-specifikt, stannar i E2E (`calendar.spec.ts`-liknande)
- Kön-select interaktion via `userEvent.click()` — shadcn/ui Select kräver portal-mounting; dropdown öppnas inte i JSDOM utan Radix-specifik setup. Tillräckligt att verifiera att triggern renderas.

## Gotchas

| Gotcha | Lösning |
|--------|---------|
| `vi.mock()` + top-level mock-objekt → hoisting-fel | Använd `vi.hoisted()` — se mönster ovan |
| `ResponsiveDialogFooter` renderar inget synligt i JSDOM | Mocka till `<div>{children}</div>` |
| shadcn/ui `Select` öppnar inte dropdown i JSDOM | Testa bara att SelectTrigger renderas (via `document.getElementById`), inte dropdown-interaktion |
| `generateUnsubscribeToken` kräver `NEXTAUTH_SECRET` | Sätt i `beforeEach`: `process.env.NEXTAUTH_SECRET = 'test-secret-for-vitest'` |
