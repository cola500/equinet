## Vad gick bra

### 1. Tre stories på under 1 timme
S31-4 (30 min), S31-5 (15 min) och S31-6 (20 min) levererades snabbt och rent. Inga blockerare, inga merge-konflikter. Rätt storlek på stories ger förutsägbar leverans.

### 2. Hook-testning fungerade direkt
Negativ test (plan utan sektion → blockeras) och positiv test (plan med sektion → passerar) båda rätt vid första körning. Skript-logiken var enkel nog att inte kräva iteration.

### 3. Autonomt sprintflöde utan mänsklig inblandning
Alla 3 stories gick från plan → implementation → PR → merge utan att fråga Johan. Subagent-reviews skippades korrekt per trivialkriterierna. Processen fungerade som designad.

### 4. Aktualitetsverifiering korrekt tillämpat
S31-4 och S31-5 innehöll `## Aktualitet verifierad` i sina planer (och S31-6 likaså), vilket demonstrerar den nya funktionen direkt vid skapandet.

## Vad kan förbättras

### 1. Husky hooks var inte executable
Pre-commit och pre-push hooks ignorerades med "not set as executable"-varning under hela sessionen. Hooken testades manuellt och fungerade, men kördes aldrig automatiskt via git commit. En `chmod +x .husky/*` saknas troligen sedan setup.

**Prioritet:** MEDEL -- hooks är ett viktigt skyddslager men kördes manuellt under sessionen med check:all.

### 2. Embedded git repository-varning (worktrees)
`git add -A` plockade upp `.claude/worktrees/angry-murdock` som en embedded git repo. Krävde manuell unstage (`git rm --cached`). Worktree-kataloger bör vara i `.gitignore`.

**Prioritet:** LAG -- mer irritation än risk, men bör fixas för att hålla git status rent.

### 3. `docs/plans/` fortfarande ~28 filer
Storyn siktade på "max ~20 filer" men landade på 28. Icke-sprint-planer (2026-02-28-*, bdd-payment-refactor.md, native-ios-analysis.md, osv.) är behållna per spec men bidrar till bruset. Dessa äldre icke-sprint-filer kan arkiveras separat vid ett tillfälle.

**Prioritet:** LAG -- 28 vs 20 är acceptabelt, inget som blockerar.

## Patterns att spara

### Hook-test-mönster för bash-scripts
Testa bash-hooks manuellt med temporära filer:
```bash
# Negativt test
cp /tmp/test-fil.md docs/plans/s99-test.md
git add docs/plans/s99-test.md
bash scripts/check-script.sh; echo "Exit: $?"
git rm -f docs/plans/s99-test.md

# Positivt test -- samma mönster med korrekt innehåll
```
Undviker att faktiskt köra en hel commit-cykel för att testa hooken.

### git mv för massflyttning av filer
```bash
git mv docs/plans/s[3-9]-*.md docs/archive/plans/
git mv docs/plans/s1[0-9]-*.md docs/archive/plans/
git mv docs/plans/s2[0-8]-*.md docs/archive/plans/
```
Glob-patterns med `git mv` fungerar direkt i bash. Git spårar rename-historik via `git log --follow`.

## 5 Whys

### Problem: Embedded worktree dök upp i git add -A

1. Varför? `.claude/worktrees/angry-murdock` är ett git-repo inuti huvudrepot.
2. Varför? Worktree skapades av ett tidigare Agent-anrop med `isolation: "worktree"`.
3. Varför? Worktree-katalogen rensades inte automatiskt när agenten var klar.
4. Varför? `.claude/worktrees/` saknas i `.gitignore`.
5. Varför? Vi har inte haft ett systematiskt behov av detta förrän worktree-agenter började användas (S25+).

**Åtgärd:** Lägg till `.claude/worktrees/` i `.gitignore`.
**Status:** Att göra (5 minuter).

Nästa steg är att mäta om förändringarna faktiskt biter -- "Verktyg använda"-sektionen ger data efter 10 stories.
