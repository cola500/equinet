# Sprint 28 Retrospektiv: Offline PWA -- stabilisering och expansion

**Tema:** Fanga all offline-flakiness systematiskt, gor E2E-offline till del av CI, stark iOS offline.
**Tid:** 2026-04-17 (2 sessioner, ~1 dag total)

---

## Levererat

| Story | Levererat | Effort |
|-------|-----------|--------|
| S28-1 Spike | Flaky-rapport: 5 deterministiska failures, rotorsak `networkidle` + SWR-polling. 4 potentiella race conditions identifierade. | ~2h |
| S28-2 CI-smoke | `offline-smoke` jobb i quality-gates.yml, 10 offline E2E-tester i CI, branch protection. | ~3h |
| S28-3 Fix rotorsaker | 10/10 offline E2E grona. 6 rotorsaker fixade: CSP, Serwist reloadOnOnline, networkidle, saknad setup dependency, fel toast-text, login-redirect. | ~4h |
| S28-5 iOS offline | 3 forbattringar med TDD (3 nya tester, 283 totalt grona): stale cache vid cold start, retry vid app resume, offline-felmeddelande. | ~2h |
| S28-6 Docs | offline-pwa.md uppdaterad med iOS, CI, flaky-learnings. 5 nya offline-learnings. | ~30 min |

**S28-4** (kund-offline) parkerad i backloggen -- sprinten levererar fullt varde utan den.

**Totalt:** 4080 unit-tester + 283 iOS-tester + 10 offline E2E + 25 smoke E2E grona.

---

## Vad gick bra

1. **Spike forst (S28-1) var ovardelig.** Alla 5 failures hade SAMMA rotorsak (`networkidle`). Utan spike hade vi jagat 5 separata buggar. 5 Whys-metoden avslojde att de INTE var flaky -- de var deterministiskt trasiga.

2. **CI-integration (S28-2) ger permanent varde.** Offline-tester som bara kors manuellt "gloms bort". Nu fangar CI regressioner automatiskt vid varje PR. Serwist-uppgraderingar kraver inte langre manuell verifiering.

3. **iOS-storyn gynnades av ios-expert plan-review.** Thread-safety-risken med `static var isOffline` fangades i plan-review (5 min) istallet for efter implementation (30+ min att refaktorera).

4. **TDD for iOS offline.** RED-GREEN-cykeln var snabb (~20 min totalt). Testerna definierade onskad beteende tydligt -- implementationen var bara att lasa testerna och koda.

5. **Docs-story sist i sprinten.** All information fran S28-1 till S28-5 var tillganglig nar docs skrevs -- ingen gissning, bara sammanfattning.

---

## Vad som inte fungerade

1. **Branch-namnforvirring (S28-5).** Feature branch skapades med ratt namn men committar hamnade pa fel branch (`feature/patterns-catalog`). Troligen en pre-existing branch som inte rensades. Lost med rename men kostade 5 min felsökning. **Atgard:** `git branch --list 'feature/*'` FORE `git checkout -b`.

2. **Manuell simulator-verifiering hoppades over (S28-5).** Storyn kraver "verifierad manuellt i simulator" men vi testade bara via kod + unit-tester. Offline-natverksavstangning i simulator kraver manuell interaktion som inte kan automatiseras. **Atgard:** Acceptera begransningen -- unit-tester tacker logiken, manuell verifiering ar nice-to-have.

3. **Ingen E2E for iOS offline.** iOS-offline-logiken testas bara med XCTest (ViewModel-niva). Ingen end-to-end-verifiering att NWPathMonitor -> banner -> stale cache -> retry-kedjan fungerar i simulator. Skulle behova mobile-mcp for fullstandig verifiering.

---

## Processandring till nasta sprint

1. **Kor `git branch --list 'feature/*'` fore ny branch.** Rensa stale branches fore ny story.

2. **Plan-review med ios-expert ar bevisat vardefullt.** Fortsatt anvanda for alla iOS-stories -- det fangar arkitekturproblem tidigt.

3. **Offline E2E ar nu del av CI.** Framover: offline-regressioner fangas automatiskt. Ingen manuell `npm run test:e2e:offline` behovs infor merge.

4. **ignoreTTL-monstret bor appliceras pa fler ViewModels.** BookingsViewModel och CalendarViewModel anvander SharedDataManagers cache men skickar inte `ignoreTTL` vid offline. Lagg till i nasta iOS-sprint.
