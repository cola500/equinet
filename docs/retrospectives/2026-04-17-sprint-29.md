# Sprint 29 Retro: iOS Polish + mobile-mcp

**Totalt:** 6 stories, ~5.5h effektiv tid.

**Nya tester:** 11 nya iOS-tester (4 NetworkMonitor, 3 OfflineVerification, 4 OfflineE2E).

**Nya filer:**
- `OfflineE2ETests.swift` -- fullstandigt E2E-scenario
- `OfflineVerificationUITests.swift` -- polling + ViewModel-integration
- `scripts/ios-offline-verification.sh` -- headless offline-verifiering

## Vad gick bra

- **UserDefaults-polling-losningen** blev elegant. `simctl spawn defaults write` ar det enda sattet att injicera state i en korrande simulator-app utan system-dialoger. Pollar varje 1s -- acceptabel overhead.
- **Debug-autologin** (`--debug-autologin`) loste SecureField-problemet helt. Mobile-mcp/WebDriverAgent kan inte interagera med SecureField (osynligt i accessibility tree), men launch arguments fungerar perfekt.
- **`#if DEBUG` guards overallt** -- noll paverkan pa release-builds. Alla debug-mekanismer (override, polling, autologin) ar helt osynliga i produktion.
- **S29-3 och S29-4 redan fixade** -- snabb verifiering med grep istallet for onodigt arbete. Bra att sprints kan rensa backlog-poster.
- **3/3 grona E2E-korningar** -- inte flaky. Generosa timeouts + explicit cache-hantering i setUp/tearDown.

## Vad som inte fungerade

- **URL-scheme-approach (reverterad)**: Forsta forsok att trigga offline via `equinet://debug-offline` kraschade pa system-dialog ("Oppna i Equinet?") som blockerar all automation. ~30 min forlorad tid innan pivot till UserDefaults.
- **WebDriverAgent-timeout vid kallstart**: Forsta screenshot-forsok timeotar ofta efter simulator-boot. Kraver reboot (`simctl shutdown` + `simctl boot`) + vantetid.
- **iOS Simulator delar natsverksstack**: Det finns inget `simctl`-kommando for att stanga av natverk per simulator. `simctl status_bar --data-network` andrar bara visuell ikon, inte faktiskt natverkstillstand. Hela debug-override-mekanismen behovdes for att krinnga detta.
- **tsconfig.typecheck.json och ios build-artifacts**: Xcode DerivedData (Supabase Swift SDK) innehaller `.ts`-filer som bryter `npm run typecheck`. Fixades med `"ios/**"` i exclude, men var ovantad.

## Processandring till nasta sprint

1. **Verifiera backlog-poster mot faktisk kod FORST**: S29-3 och S29-4 var redan fixade. Gor en snabb `grep` innan du skriver plan for mekaniska fixar.
2. **Prova UserDefaults-injection fore URL-scheme**: For debug-automatisering i iOS Simulator ar UserDefaults + polling alltid battre an URL-scheme (inga system-dialoger).
3. **Mobile-mcp beslutstrald dokumenterad**: ios-learnings.md har nu en tabell for nar XCTest vs mobile-mcp vs shell-skript vs XCUITest. Folj den.
4. **CI-beslut**: iOS offline-tester kors lokalt pre-release. Kan laggas till i CI om self-hosted macOS runner satts upp.
