## Vad gick bra

1. **Snabb leverans**: 5 stories på ~2 sessioner. Cleanup + research-stories (S16-1, S16-2, S16-5) var redan klara vid start.

2. **TDD-disciplin**: businessName-fixen i S16-3 och hela admin-härdningen i S16-4 följde RED-GREEN-REFACTOR konsekvent. Testerna hittade buggar direkt (Prisma mock utan mockResolvedValue).

3. **Code review som kvalitetsgate**: Två code reviews (S16-3 + S16-4) hittade reella problem -- whitespace-only businessName, duplicerad const, Zod saknas på query params. Alla fixade innan merge.

4. **withApiHandler-utvidgning**: `auth: "admin"` med automatisk audit-loggning är en ren, centraliserad lösning. Nya admin-routes får audit gratis.

## Vad kan förbättras

1. **RLS-divergens lokalt vs Supabase**: GET /api/services (migrerad i S14) returnerar ALLA tjänster lokalt utan RLS. E2E-test för tjänsteskapande fick skippas. Behöver antingen lokal RLS-setup eller RLS-aware testmiljö.

2. **Lokal Docker saknar triggers**: `handle_new_user`-triggern finns inte i Docker. E2E-seeding behöver fallback. Bör lösas permanent (kanske via `prisma migrate dev` med anpassade migrationer).

3. **Dual admin-auth**: Två system finns nu (`admin-auth.ts` med DB-lookup + `roles.ts` med session-check). Befintliga admin-routes bör migreras till `withApiHandler({ auth: "admin" })` opportunistiskt.

4. **Migrationsstatus lokalt**: 5 migrationer var inte applicerade lokalt (RLS-policies kräver Supabase auth-schema). Behöver bättre rutiner för att hålla lokal DB i synk.
