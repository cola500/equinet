## Vad fungerade inte

**Plan-commit på main av misstag (andra gången):**
`docs/plans/s44-2-plan.md` committades direkt till main istället för feature branch. Skapade divergent branches, PR #230 kunde inte fast-forward-mergas. Löstes med `git pull origin main --no-rebase` + `git push`. PR hade redan mergiats av CI.

Obs: commit-strategy.md tillåter plan-filer direkt på main — felet var inte att det committades på main utan att det skapade divergens. Workaround: om plan-commit görs på main FÖRE feature branch skapas, skapa branch från main efter plan-commit.

**E2E-smoke: 1 pre-existerande failure:**
`auth.spec.ts` "should register a new provider" failar pga lokalt Supabase-state (registrering navigerar ej till `/check-email`). Inte relaterat till S44 — ingen auth-kod rördes.

## Processändring

**Ingen ny processändring behövs.** Plan-commit-divergens är känt problem (inträffat 2 gånger). Commit-strategin är korrekt — implementationsordningen behöver justeras:

> Skapa feature branch FÖRE plan-commit om planen ska leva på main. Alternativt: committa planen på feature branchen och merga med `--no-ff`.

Detta är en reminder, inte en regeländring.
