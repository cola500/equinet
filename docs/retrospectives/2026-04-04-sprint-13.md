# Sprint 13 Retro: Supabase Auth Fas 3 -- Cleanup

**Sprint:** 13
**Period:** 2026-04-03 -- 2026-04-04
**Goal:** Ta bort NextAuth. En auth-kalla. RLS automatiskt.

| Story | Beskrivning | Commit |

**6/6 stories klara.** Sprint-malet uppnatt.

## Vad gick bra

- **Ren avveckling**: NextAuth, MobileTokenService, passwordHash och bcrypt borttagna utan regressioner
- **Visuell verifiering bekraftade**: 12 webb-sidor kontrollerade via Playwright MCP -- alla fungerar
- **Testlosenord overlevde migrering**: bcrypt-hash kopierades korrekt fran public.User till Supabase auth.users
- **Lokal dev-setup**: Lokal PostgreSQL + Supabase Auth (remote) fungerar smidigt

## Vad som inte fungerade

- **iOS-verifiering kunde inte goras automatiskt**: mobile-mcp kraver en iOS-build mot aktuellt Supabase-projekt, vilket kraver manuell Xcode-setup
- **Seed-scripts skapar inte Supabase Auth-anvandare**: Seeding gor bara public.User, inte auth.users -- nya utvecklare maste kora migreringsscriptet manuellt

## iOS-verifiering -- separat story kravs

S13-6 verifierade webben men INTE iOS. iOS-appen (Supabase Swift SDK, S13-4) kraver:

1. **Ny Xcode-build** mot Supabase-projektet `zzdamokfeenencuggjjp`
2. **Manuell verifiering** i simulator eller fysisk enhet:
   - Login via Supabase Swift SDK
   - Dashboard med data
   - Navigation: alla tabs
   - WebView-sidor: autentiserade (cookies)
   - Annonsering, Insikter (native skarmar)
   - Logout + re-login
3. **Eventuella fixar** om nagot inte fungerar

**Rekommendation:** Lagg till som story i nasta sprint (eller backlog).
Johan tar med Lead for prioritering.

## Processandring till nasta sprint

- **Seed bor skapa Supabase Auth-anvandare**: Overvagg att uppdatera seed-scriptet sa att det aven skapar anvandare i Supabase Auth (via admin.createUser). Forenklar onboarding for nya utvecklare.
- **iOS-verifiering bor vara en del av merge-kriterier**: Nar iOS-stories mergas bor det finnas en verifieringsplan som inte beror pa manuell Xcode-setup.

## Auth-migrering komplett?

**Ja, for webb.** En auth-kalla (Supabase Auth), NextAuth borta, passwordHash borta.

**Aterstar for iOS:**
- Manuell verifiering av Supabase Swift SDK-integration
- Eventuella bugfixar som upptacks vid verifiering

**Aterstar generellt:**
- Seed-scripts for Supabase Auth-anvandare (nice-to-have)
- RLS-policies per tabell (paborjat i S10/S11, behover utvidgas)
