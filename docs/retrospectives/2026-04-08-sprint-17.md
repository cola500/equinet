## Vad gick bra

1. **Quick wins levererade snabbt.** S17-1/2/3 kunde batchas i en enda session (45 min totalt). Visar att sma, valgransade stories ar effektiva.

2. **supabase start-bytet (S17-7) var ratt tajming.** Loser tre problem fran sprint 16-retron (RLS-divergens, saknade triggers, migrationer ur synk). Nu ar lokal dev identisk med prod-miljon.

3. **Admin-routes-migreringen (S17-8) var rent mekanisk.** -421 rader, centraliserad rate limiting + audit log pa koopet. Parallella agenter fungerade bra for batchmigrering.

4. **Edge Config (S17-6) ger reell prestandavinst.** Feature flag-lasning gick fran ~50ms (PostgreSQL + 30s cache) till <1ms (globalt Edge Config). DB-fallback gor det robust.

5. **iOS miljohantering (S17-9) okar testbarhet.** Enkelt att vaxla mellan lokal dev och staging via Xcode launch argument.

---

## Vad gick mindre bra

1. **S17-5 (Resend e-post) blev aldrig startad.** Kraver manuell verifiering i prod (skicka testmail, kolla SPF/DKIM). Inte latt att automatisera. Ratt att flytta till backlog -- men bor prioriteras fore lansering.

2. **WAF-regler (S17-2) ar manuella.** Kan inte versionshanteras eller automatiseras pa Hobby-plan. Dokumentation ar basta vi kan gora just nu.

3. **Edge Config token-granularitet saknas pa Hobby.** Inga skrivskyddade tokens -- alla tokens har full access. Acceptabelt for nu men bor fixas vid Pro-uppgradering.

## Laerdomar

### Tekniska

- **`supabase start` ersatter Docker for lokal dev** -- men `prisma migrate dev` fungerar INTE (shadow DB saknar auth-schema). Använd `prisma migrate deploy` istallet.
- **`.env.local` trumfar `.env`** -- Vercel CLI:s `env pull` skapar `.env.local` som overstyrer. Uppdatera BADA vid DB-byte.
- **Edge Config Free: 1 store, 8KB max** -- racker for ~100 feature flags. Ingen granular token-scoping pa Hobby.
- **pg_cron: VACUUM fungerar inte pa Supabase managed** -- Supabase kor auto-vacuum. Använd bara for datarensning.
- **iOS `URL(string:relativeTo:)` over `appendingPathComponent`** -- det senare URL-encodar `/` i strangar.
- **`vi.mock()` maste komma fore imports** (hoisting) -- annars mockas inte modulen.

### Process

- **Batchning av sma stories fungerar.** S17-1/2/3 i en session var effektivt. Forutsattning: oberoende stories, minimal risk.
- **Mekanisk migrering med parallella agenter** skalade bra (S17-8: 13 routes, -421 LOC, 0 regressioner).
- **Manuella prod-verifieringar bor planeras explicit.** S17-5 blev kvar for att den kravde manuellt arbete som ingen allokerade tid for.

## Processforandringar till nasta sprint

1. **Tagga manuella stories tydligt.** Stories som kraver manuell verifiering (prod-test, dashboard-konfiguration) bor markeras med `manual` sa de inte fastnar i automationsfloden.
2. **Pro-uppgradering bor prioriteras.** Flera begransningar (WAF-automation, Edge Config tokens, timeout-granser) forsvinner med Vercel Pro. Blocker for lansering.
3. **Resend-verifiering fore lansering.** S17-5 ar HOG i backlog -- schemalag manuell session med Johan for prod-test av e-postflodet.
