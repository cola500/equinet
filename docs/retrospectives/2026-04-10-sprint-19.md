# Sprint 19 Retro: E2E Test Hardening

## Levererat

- **8 stories**, alla klara i en session
- **2 spec-filer borttagna** (stripe-payment.spec.ts, flexible-booking.spec.ts)
- **39 waitForTimeout eliminerade** i 3 specs (calendar 10, route-planning 8, announcements 21)
- **3 externa specs** separerade från standard-sviten (customer-insights, offline-mutations, offline-pwa)
- **supabase/seed.sql** skapad med auth triggers för lokal E2E-bootstrap
- **npm run test:e2e:bootstrap** + **npm run test:e2e:external** -- nya scripts
- Netto: 36 spec-filer (ner från 38), ~53 kvarvarande waitForTimeout (ner från ~92)

## Vad gick bra

- **Snabb exekvering**: Alla 8 stories klara i en session utan blockerare
- **waitForTimeout-ersättningar**: Promise.race-mönstret (data ELLER empty state) fungerade utmärkt
- **Triviala stories (S19-1)** ger momentum och låter nästa stories bli mer fokuserade
- **Sprint-docen var tydlig** med prioritetsordning och exekveringsplan
- **Pre-push hook** fångade ett flaky test (error.test.tsx) -- inte relaterat till våra ändringar, passerade vid retry

## Vad som inte fungerade

- **Sprint-docen underskattade waitForTimeout-antal**: Sa 6/5/8 men det var 10/8/21. Räkna alltid själv.
- **Flaky test blockerade push**: `src/app/provider/error.test.tsx` "shows retry button that calls reset" failade i full svit men passerade isolerat. Borde undersökas separat.
- **53 kvarvarande waitForTimeout**: Scope-avgränsningen var rätt (3 specs), men ~53 kvarstår i booking, provider, no-show, recurring-bookings mfl. Potentiellt nästa sprint eller opportunistiskt.

## Processändring till nästa sprint

- **Flaky test-undersökning**: error.test.tsx bör fixas -- trolig timing-issue med `act()` wrap
- **Kvarvarande waitForTimeout**: Prioritera booking.spec.ts (9 st) och provider.spec.ts (8 st) näst
- **E2E-körning**: Bör köra full E2E-svit som sprint-gate, men kräver lokal Supabase + dev-server igång
