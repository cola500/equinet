# Sprint 37 Retro: Messaging-rollout

**Datum:** 2026-04-18
**Modell:** sonnet
**Total effort:** ~2h (S37-1: 30 min, S37-2: 1h, S37-3: 15 min)

## Levererat

| Story | Beskrivning | Status |
|-------|-------------|--------|
| S37-1 | ThreadSkeleton ersätter `fallback={null}` — eliminerar blank flash | Mergad PR #209 |
| S37-2 | customerName/serviceName från API (XSS/injection-risk eliminerad) | Mergad PR #210 |
| S37-3 | `messaging: defaultEnabled: true` + rollout-docs | Mergad PR #211 |

**Tester:** 4165 unit-tester (+ 1 nytt test för serviceName null-fallback + uppdaterade assertions)
**E2E smoke:** 25/25 pass, 3 skip

## Vad gick bra

- **Liten sprint, tydlig scope.** 3 stories med explicit avgränsning — inga scope-diskussioner.
- **S37-2-strategin.** Att lägga `customerName`/`serviceName` i befintligt GET-svar (istället för ny endpoint) var rätt — ThreadView SWRar redan det endpointet. Noll extra nätverksförfrågan.
- **Security-reviewer fångade Minor.** GET rate-limit saknade try/catch för `RateLimitServiceError` — fixat i samma story. Pattern-inkonsistens fångad och åtgärdad.
- **TDD-flödet.** RED-tester skrivna INNAN implementation för S37-2 — tydlig bekräftelse att testerna faktiskt testar rätt sak.

## Vad som inte fungerade

- **E2E lock-fil.** Stale `.next/dev/lock` blockerade E2E smoke. Enkel fix (rm lock) men borde inte hända. Beror troligen på en avbruten dev-server-session.
- **git add -A inkluderade screenshots.** Committade `screenshots/`-katalogen oavsiktligt i S37-2. Ingen skada men onödigt brus i historiken.

## Processändring till nästa sprint

- Vid sprint-avslut: kör alltid `rm -f .next/dev/lock` INNAN `test:e2e:smoke` för att undvika låsblockeringar.
- Lägg till `screenshots/` i `.gitignore` om det inte redan finns.

## Metrics

- **Modell:** sonnet genomgående
- **Effort vs plan:** 2h (exakt som plan)
- **Reviews:** security-reviewer för S37-2 (inga blockers, 2 minors varav 1 fixad), triviala skippade
- **Audit-fynd stängda:** 2 MAJOR från S36-2 (MAJOR-1: skeleton, MAJOR-2: injection) → 0 MAJOR kvar
- **Messaging-status:** Aktiverad för alla användare. Rollback via admin-toggle.
