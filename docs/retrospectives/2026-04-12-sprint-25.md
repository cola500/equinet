## Vad gick bra

- **check:all 4/4 grön** vid varje kontrollpunkt.
- **En commit, en PR** -- rent flöde.
- **Docs-stories trivial att göra direkt** -- ingen vinst av att delegera.

## Vad som inte fungerade

### Worktree-agent blockerad

Sonnet-agenten i worktree blockerades av:
- **Write nekades** för `.claude/rules/`-filer
- **Bash nekades** -- kunde inte köra git commit
- **Resultat:** 71k tokens, 4.5 min, 0 commits

Agenten skapade en sessionsfil i `docs/` (Write tillåts där) men inget annat.

### Overhead > vinst för docs-stories

Docs-stories tar 5-10 min att göra direkt. Spawn + vänta + merge + cleanup kostar nästan lika mycket. Netto tidsvinst: noll eller negativ.

## Analys av worktree-agent-mönstret

### Var det INTE funkar

| Scenario | Problem |
|----------|---------|
| Docs/config-stories (5-10 min) | Overhead > vinst |
| Stories som rör `.claude/rules/` | Write blockeras |
| Stories som kräver git-kommandon | Bash blockeras |
| Triviala ändringar | Inte värt att koordinera |

### Var det POTENTIELLT funkar

| Scenario | Förväntad vinst | Krav |
|----------|----------------|------|
| Två tunga webb-stories i separata domäner (30+ min vardera) | 30-50% tidsvinst | Rättigheter lösta, inga delade filer |
| Mekanisk batchändring (rename, migrera pattern) | Parallellt med annan story | Tydlig scope, ingen bedömning krävs |
| Webb + iOS parallellt | Helt separata filträd | iOS-testning fungerar i worktree |

### Rättighetsfrågan

Worktree-agenter ärver INTE samma rättigheter som huvudsessionen. Användaren promptas att godkänna varje verktygstyp -- men i bakgrundsagenter finns ingen interaktiv prompt, så verktyget nekas direkt.

**Möjlig lösning:** Konfigurera allowlist i `.claude/settings.json` för worktree-paths. Ej testat.

## Rekommendationer för S26

### 1. Testa subagenter UTAN worktree först

Spawna `Agent(model: "sonnet", run_in_background: true)` (utan `isolation: "worktree"`) för research/analys-uppgifter som inte behöver skriva filer:
- Kör code-reviewer i bakgrunden medan du implementerar nästa story
- Kör security-reviewer parallellt med cx-ux-reviewer
- Samla information (grep, read, analyze) för planering

**Hypotes:** Subagenter utan worktree har fulla rättigheter och kan köra parallellt med huvudsessionen.

### 2. Testa worktree med en riktig 30+ min story

Om rättigheterna kan lösas: välj två stories i helt separata domäner (t.ex. `useProviderCustomers.ts` refactoring + en ny API route) och kör parallellt.

### 3. Strukturera S26 som A/B-test

| Story | Approach | Mäts |
|-------|----------|------|
| S26-1 | Direkt (baseline) | Tid, tokens |
| S26-2 | Bakgrundsagent review (parallell) | Tid, tokens, review-kvalitet |
| S26-3 | Worktree-agent (om rättigheter lösta) | Tid, tokens, merge-friktion |

Mät: total sessionstid, tokens förbrukade, antal blockerings-incidenter, kvalitet på output.
